What is Lelylan Types

Lelylan Types is a place where you can define and discuss the structure of any device type like lights, locks, cameras, sensors and more. Why? Because to create an open connected home where everything communicates we need an open and clear way to define how these devices work.

The Pain

Right now we have dozens of standards: Wifi, Zigbee, Ethernet, Konnex, X10 to name some. If you want to let every device communicate between each other you will have hard time. This is one of the reasons why the connected home is not yet a reality.

The Solution

Our solution to this problem is to create a type, which in its simplest form can be defined as the combination of three key elements:

  • Properties - what vary during time
  • Functions - what a device can do
  • Statuses - what a device is in a specific moment

A Simple Example

The simples example is the Basic Light which can be defined as follow.

  • Properties: status accepting values on and off
  • Functions: turn on and turn off
  • Statuses: light on and light off

Do you want to see some? Search between some of the existing types.

How can I use a type

Types are not used only to define a device structure. You need them to let the physical world communicate with the web. If you are into connecting physical devices to the web read the article Building the Connected Light.

Ad Hoc Types

Although we are trying to define a standard set of types we are aware of the fact that every need can be different. For this reason everyone can create a new type from scratch to better satisfy your own needs.
