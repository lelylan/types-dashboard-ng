How to create a type

To create a new type you need to click "create type". If signed in, a form asking you for a name and a description will show up.

Properties

Once you have filled the required information you are asked to create the type properties. For a light they could be status, intensity, color and blinking times.

To add a new property click "create new property". A modal form will appear asking for the following params.

  • Name - A descriptive name for the property.
  • Type - Accepted values are text (string values), number (number values), range (number values inside a specified range) and color (#000000 hex value).
  • Default - Value assigned to the device property when initialized.
  • Suggested - List of value/description pairs where value is the value used to update the device properties and description is a friendly description associated to the value (useful for UI definitions). This field is visible when type is not set to range.
  • Range - Accepted values defined between a min, max and step value. This field is visible when type is set to range.

To better understand how properties are defined check out some types in popular or check out Lelylan Properties API (in private beta).
